# Document Upload Validation And Quota Policy

> Status: Active baseline (Prompt 23)
> Scope: Server-side file validation, quota enforcement, and safe failure behavior

This policy defines admission controls for `POST /projects/{projectId}/documents/import`.

## Allowed file families

Accepted extensions:

- `.pdf`
- `.tif`
- `.tiff`
- `.png`
- `.jpg`
- `.jpeg`

Accepted detected types:

- `application/pdf`
- `image/tiff`
- `image/png`
- `image/jpeg`

## Validation controls

Validation is enforced server-side and does not trust client MIME headers.

Controls:

1. Extension allowlist check.
2. Magic-byte detection from the upload payload.
3. Extension-to-signature match requirement.
4. Maximum upload byte limit (`DOCUMENTS_MAX_UPLOAD_BYTES`).
5. Project byte quota check (`DOCUMENTS_PROJECT_QUOTA_BYTES`).

Failure posture:

- Validation failures fail closed.
- Unsupported or mismatched signatures do not enter scan/extraction.
- Quota failures happen before scan handoff.

## Quota semantics

Project quota is byte-scoped and evaluated as:

- current project byte usage (non-canceled records)
- plus incoming upload size
- must be less than or equal to `DOCUMENTS_PROJECT_QUOTA_BYTES`

If exceeded, the upload is rejected with explicit quota-failure messaging.

## Storage key generation

Storage keys are generated by the API and never use user-provided path values:

- `controlled/raw/{project_id}/{document_id}/original.bin`
- `controlled/raw/{project_id}/{document_id}/source-meta.json`

This preserves controlled-path policy and prevents executable/path-manipulation behavior.

## Final integrity verification

After moving the upload into controlled storage, the API re-reads `original.bin` and verifies:

- persisted byte count equals streamed byte count
- persisted SHA-256 equals streamed SHA-256

If verification fails, the import is marked `FAILED` and does not proceed.

## Environment variables

- `DOCUMENTS_MAX_UPLOAD_BYTES` (default `67108864`)
- `DOCUMENTS_PROJECT_QUOTA_BYTES` (default `536870912`)
- `DOCUMENT_SCANNER_BACKEND` (`auto | stub | none`)
- `STORAGE_CONTROLLED_ROOT` (local controlled storage root for the API process)
